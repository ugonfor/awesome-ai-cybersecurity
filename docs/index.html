Taxonomy

Security OF AI

Protecting the AI model itself

Jailbreak defense, prompt injection robustness

Security BY AI

AI performing cybersecurity tasks

CTF solving, vuln analysis, SOC automation

Security FROM AI

Assessing risks posed by AI

Malware gen capability, exploit writing risk

Key Terms

pass@kSuccess rate with k attempts. Higher k inflates scores.
CTFCapture The Flag — competitive cybersecurity.
ASLAI Safety Level (Anthropic). ASL-3 = expert exploit dev.
CCLCritical Capability Level (Google). L1 = 10x cost cut.
HIGHOpenAI risk. End-to-end ops or zero-day discovery.

Why cross-company comparison is hard

  • Different benchmarks: OpenAI uses CVE-Bench, Anthropic uses Cybench/CyberGym, Google uses Key Skills, xAI uses WMDP-Cyber.
  • Different agent setups: Codex CLI vs Terminal Tool vs Custom controller vs UK AISI Inspect.
  • Different pass@k: Anthropic uses p@1 or p@10; OpenAI uses p@12; xAI is unguided.
  • Different difficulty: Each company's "Professional CTF" is a different challenge set.
