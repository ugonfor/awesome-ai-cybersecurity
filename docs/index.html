Awesome AI Cybersecurity

Curated AI cybersecurity benchmarks, agents & tools — organized by Security OF / BY / FROM AI.
The AI cybersecurity space is fragmented: OpenAI, Anthropic, Google, and xAI each use different benchmarks, and terms like “red team,” “safety,” and “security” are used inconsistently. This site cuts through the confusion with a clear taxonomy and comprehensive cross-comparison.

Taxonomy

Axis 1: Whose security?

Security OF AI

Protecting the AI model itself

Jailbreak defense, prompt injection robustness

Security BY AI

AI performing cybersecurity tasks

CTF solving, vulnerability analysis, SOC automation

Security FROM AI

Assessing risks posed by AI

Malware generation capability, exploit writing risk

Axis 2: Offensive vs Defensive

CategoryMeaning
Offensive (Red Team)Attack simulation, vulnerability discovery
Defensive (Blue Team)Detection, analysis, incident response
EvaluationPure capability measurement

Key Terms

pass@kSuccess rate with k attempts. pass@1 = first try; pass@30 = at least 1 in 30. Higher k inflates scores.
CTFCapture The Flag — competitive cybersecurity challenges (web, RE, crypto, pwn).
ASLAI Safety Level (Anthropic). ASL-3 = model can assist expert exploit development.
CCLCritical Capability Level (Google). Cyber Uplift L1 = 10x cost reduction for attacks.
HIGHOpenAI risk level. Can automate end-to-end ops against hardened targets or discover zero-days.
